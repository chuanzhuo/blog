2023年12月：支持m3u链接、m3u源文件等IPTV源检查器

打开IPTV Checker，在线M3U8播放器 或 观看IPTV直播，测试M3U直播源
若订阅源网址无法打开，将网址粘贴到 在线网络加速器 或 在线浏览器 打开，然后全选复制订阅源内容

步骤（干货）

  1. 浏览器打开：http://tonkiang.us
  2. 输入‘电台，电视台’的名称后（比如bbc），回车
    • 可以先去了解下，你想要观看的电视台的名称，各国相应的语言
  3. 在搜索结果页面的左侧 菜单中，点击‘播放’按钮
    • 这是一个列表，该列表尝试给出各个播放源的地址，如果你选择的播放源有问题，可以尝试切换至其他播放源
  4. 点击M3U8 IPTV PLAYER 的播放按钮。
    • 在弹出的新的页面中（这时M3U8播放器开始加载直播流，有时你需要点击一下播放按钮，告诉播放器请开始播放），比如：http://zqjy.info/t/Njc4ODI2

我们相信会有更多的直播源搜索器出现，这时人们开始更加关注：播放媒体（电视节目）的质量。
实测中发现：各个源中的网速很佛系，有的连接速度快，很多都慢而且卡顿；而卡顿对于电视电影节目是很难忍受。

背景资料

偶然的机会，在研究学习VLC播放器的时候，发现了这么一个神器（http://tonkiang.us），估计它的后台爬虫类似于谷歌那样爬取M3U和M3U8的url地址（或者某种机制提交直播流地址到特定的数据库中）。

显然它需要解决一些问题

  1. 不停的轮询（定时）验证m3u8或者m3u文件中的地址有效性
    • 这是一个我刚实践过的项目。github上搜索iptv-checker，当然后来我没有成功在Mac M1上编译成功，而是使用了brew install iptv-checker来进行已编码可执行文件的安装。
    • 在命令行中输入：iptv-checker **.m3u 
    • 在当前目录下，这个命令会生成一个文件夹，将m3u中的每一个m3u8链接进行有效性验证，验证可得online的所有m3u8电台
    • 将这个有效且online的m3u文件，直接拉到VLC中，即可播发里面的流媒体了。
  2. 对直播源地址的信号中转（专业上称为：推流）
  3. 对无法直达直播地址的魔法访问，比如通过proxy来推流‘网外’的直播流
    • 这也是我学到的一个认知盲区：VLC是可以通过代理来用访问流媒体的。在VLC的配置里面（有点难找，简单说：在 Preference(show all) > Input/Codesc > Socks Proxy）具体步骤请自行搜索（也可以找我探讨）

根据提供的IPTV Checker服务分享用户提交的m3u8文件内容，可用来‘撞大运’（根据个人的网络环境和播放当时服务器及网络通信状态，播放体验比较佛系）观看些全球各地的IPTV源（双击某一行复制，粘贴到M3U8播放器播放）：

后台格式：ffprobe -v quiet -print_format json -show_format -show_streams https://streamtv.cdn.dvmr.fr/TVR/ngrp:tvr.stream_all/master.m3u8

正则表达式提取：^ffprobe -v quiet -print_format json -show_format -show_streams [\w:/.-?=@$]+$
